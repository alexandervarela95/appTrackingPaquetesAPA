1. Para que sirve el sistema

appTrackingPaquetesAPA sirve para registrar paquetes internos, darles seguimiento por numero de guia, consultar su estado, reportar problemas, subir comprobantes y revisar informacion administrativa.

Una guia es el codigo que identifica un paquete. El formato actual es APA-000001. La guia la genera el sistema; el usuario no debe escribirla manualmente.

Idea sencilla: si el paquete fuera una persona en una fila, la guia seria su numero de turno. Con ese numero se puede encontrar rapidamente en el sistema.

3. Reglas basicas de uso

  1. No comparta su usuario ni contrasena.
  2. Revise bien origen, destino, remitente y destinatario antes de guardar.
  3. Use descripciones claras. Ejemplo: Factura original de proveedor, no solo Documento.
  4. Si hay un problema, registre una incidencia.
  5. Si tiene una foto, PDF o comprobante, subalo como evidencia.
  6. Al terminar, cierre sesion desde la burbuja inferior del usuario.

4. Como iniciar sesion

  1. Abra el navegador.
  2. Entre a http://localhost:4300/login.
  3. Escriba el correo asignado.
  4. Escriba la contrasena.
  5. Presione el boton de inicio de sesion.
  6. Si los datos son correctos, vera la pantalla de inicio o dashboard.

5. Como leer el dashboard

Para una persona operativa, esta pantalla responde: cuantos paquetes hay, cuantos problemas existen y si hay comprobantes cargados.

Para gerencia, esta pantalla sirve como vista rapida de volumen de trabajo, incidencias pendientes y actividad general.

6. Paquetes

  1. Entre al menu Paquetes.
  2. Revise la lista de paquetes registrados.
  3. Para buscar un paquete, escriba una guia como APA-DEMO-2026.
  4. Use Ver detalle para revisar toda la informacion.
  5. Use Seguimiento para ver los movimientos del paquete.

7. Registrar un paquete

  1. Presione Registrar paquete.
  2. Seleccione tipo de paquete y prioridad.
  3. Escriba una descripcion clara.
  4. Seleccione origen y destino.
  5. Seleccione remitente y destinatario.
  6. Seleccione motorista si aplica.
  7. Presione Registrar paquete.
  8. El sistema guarda el paquete y genera su guia automaticamente.

8. Consolidar paquetes

Use esta pantalla cuando varios paquetes comparten datos, por ejemplo mismo destino y mismo destinatario. El sistema no guarda nada en MongoDB mientras usted solo agrega a la lista temporal.

  1. Seleccione los datos comunes: origen, destino, remitente, destinatario y prioridad.
  2. Escriba tipo, descripcion y observaciones del primer paquete.
  3. Presione Agregar a lista.
  4. Repita el proceso para cada paquete adicional.
  5. Revise la tabla temporal.
  6. Si un dato esta mal, use Editar. Si no debe ir, use Quitar.
  7. Presione Guardar todos cuando la lista este correcta.
  8. El backend crea cada paquete por separado y genera una guia para cada uno.
Ejemplo: si guarda tres paquetes, el sistema puede responder con APA-000001, APA-000002 y APA-000003. Cada guia se puede buscar despues en Paquetes o Seguimiento.

9. Detalle del paquete

El detalle sirve para confirmar toda la informacion de un paquete. Desde aqui se puede revisar la guia, prioridad, origen, destino, usuarios relacionados y observaciones.

10. Imprimir hoja de envio

  1. Abra el detalle del paquete.
  2. Presione la opcion de impresion si esta disponible.
  3. Revise que la guia, origen, destino y descripcion sean correctos.
  4. Presione Imprimir.
  5. En la impresion no deben salir menu lateral ni botones del sistema.

11. Seguimiento

Seguimiento responde a la pregunta: que ha pasado con este paquete. Cada movimiento queda ordenado por fecha.

  1. Entre a Seguimiento.
  2. Escriba una guia, por ejemplo APA-DEMO-2026.
  3. Presione Buscar.
  4. Revise la tabla de movimientos.

12. Problemas

Registre una incidencia cuando algo no va normal: paquete demorado, dato incorrecto, entrega incompleta, extravio o dano.

13. Comprobantes

Use comprobantes para respaldar una entrega, un problema o una recepcion. Los archivos permitidos son JPG, JPEG, PNG y PDF, con limite de 5 MB.

17. Buenas practicas

Para usuarios operativos

  1. Escriba descripciones claras.
  2. Revise destino y destinatario antes de guardar.
  3. Use consolidacion cuando son varios paquetes para la misma persona.
  4. Suba comprobantes cuando exista evidencia util.
  5. Cierre sesion al terminar.

Para supervision y gerencia

  1. Revise dashboard e informes con frecuencia.
  2. Observe incidencias abiertas.
  3. Mantenga usuarios y ubicaciones actualizados.
  4. Pida que cada movimiento importante quede registrado.
  5. Use las guias para auditar entregas y tiempos.
