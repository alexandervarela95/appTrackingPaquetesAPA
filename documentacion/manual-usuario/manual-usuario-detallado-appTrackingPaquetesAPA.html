1. Para qué sirve el sistema

appTrackingPaquetesAPA ayuda a llevar control de los paquetes internos: quién los envía, hacia dónde van, en qué estado están, qué problema tuvieron y qué comprobante se adjuntó.

La guía es el número que identifica cada paquete. El formato actual es APA-000001. El sistema crea ese número automáticamente; el usuario no debe inventarlo ni escribirlo a mano.

Forma sencilla de entenderlo: la guía es como el número de turno del paquete. Con ese número se busca rápido y se evita confundir un paquete con otro.

3. Reglas básicas de uso

  1. No comparta su usuario ni contraseña.
  2. Revise bien de dónde sale el paquete, hacia dónde va, quién lo envía y quién lo recibe antes de guardar.
  3. Use descripciones claras. Ejemplo: Factura original de proveedor, no escriba solo Documento.
  4. Si algo sale mal, registre el problema en el sistema.
  5. Si tiene una foto, PDF o comprobante, súbalo como respaldo.
  6. Al terminar, cierre sesión desde el botón de usuario.

4. Cómo iniciar sesión

  1. Abra el navegador.
  2. Entre a http://localhost:3180/login.
  3. Escriba el correo asignado.
  4. Escriba la contraseña.
  5. Presione el botón de inicio de sesión.
  6. Si los datos son correctos, verá la pantalla de inicio.

5. Cómo leer la pantalla de inicio

Para bodega u operación, esta pantalla ayuda a saber cuántos paquetes hay, cuántos problemas se han reportado y si existen comprobantes cargados.

Para supervisión o gerencia, sirve como una vista rápida del movimiento de paquetes y de los puntos que necesitan atención.

6. Paquetes

  1. Entre al menú Paquetes.
  2. Revise la lista de paquetes registrados.
  3. Para buscar un paquete, escriba una guía como APA-DEMO-2026.
  4. Use Ver detalle para revisar toda la información.
  5. Use Seguimiento para ver los movimientos del paquete.

7. Registrar un paquete

  1. Presione Registrar paquete.
  2. Seleccione el tipo de paquete y la prioridad.
  3. Escriba una descripción clara.
  4. Seleccione origen y destino.
  5. Seleccione remitente y destinatario.
  6. Seleccione motorista si aplica.
  7. Presione Registrar paquete.
  8. El sistema guarda el paquete y crea su guía automáticamente.

8. Registrar varios paquetes juntos

Use esta pantalla cuando varios paquetes tienen datos parecidos, por ejemplo el mismo destino o la misma persona que recibe. Mientras solo agregue paquetes a la lista, todavía no quedan guardados de forma definitiva.

  1. Seleccione los datos que se repetirán: origen, destino, remitente, destinatario y prioridad.
  2. Escriba tipo, descripción y observaciones del primer paquete.
  3. Presione Agregar a lista.
  4. Repita el proceso para cada paquete adicional.
  5. Revise la lista antes de guardar.
  6. Si un dato está mal, use Editar. Si un paquete no debe ir, use Quitar.
  7. Presione Guardar todos cuando la lista esté correcta.
  8. El sistema crea cada paquete por separado y genera una guía para cada uno.
Ejemplo: si guarda tres paquetes, el sistema puede crear APA-000001, APA-000002 y APA-000003. Cada guía se busca después por separado.

9. Detalle del paquete

El detalle sirve para confirmar toda la información de un paquete. Desde aquí se revisa la guía, prioridad, origen, destino, personas relacionadas y observaciones.

10. Imprimir hoja de envio

  1. Abra el detalle del paquete.
  2. Presione la opcion de impresion si esta disponible.
  3. Revise que la guía, origen, destino y descripción sean correctos.
  4. Presione Imprimir.
  5. En la impresión no deben salir menú lateral ni botones del sistema.

11. Seguimiento

Seguimiento responde a la pregunta: qué ha pasado con este paquete. Cada movimiento queda ordenado por fecha.

  1. Entre a Seguimiento.
  2. Escriba una guía, por ejemplo APA-DEMO-2026.
  3. Presione Buscar.
  4. Revise la tabla de movimientos.

12. Problemas

Registre un problema cuando algo no vaya normal: paquete demorado, dato incorrecto, entrega incompleta, extravío o daño.

13. Comprobantes

Use comprobantes para respaldar una entrega, un problema o una recepción. Los archivos permitidos son JPG, JPEG, PNG y PDF, con límite de 5 MB.

14. Personal, ubicaciones y estados

Estas pantallas sirven para mantener ordenada la información base. Si estos datos están bien, al registrar paquetes se cometen menos errores.

15. Informes para supervisión

Los informes muestran información reunida en un solo lugar. Para gerencia, ayudan a ver cuántos paquetes se mueven, qué problemas siguen pendientes y qué actividad reciente existe.

17. Buenas practicas

Para bodega y usuarios operativos

  1. Escriba descripciones claras y fáciles de entender.
  2. Revise destino y destinatario antes de guardar.
  3. Use Registrar varios paquetes juntos cuando son varios paquetes para la misma persona.
  4. Suba comprobantes cuando tenga una foto, PDF o respaldo útil.
  5. Cierre sesión al terminar.

Para supervisión y gerencia

  1. Revise la pantalla de inicio e informes con frecuencia.
  2. Observe problemas abiertos.
  3. Mantenga usuarios y ubicaciones actualizados.
  4. Pida que cada movimiento importante quede registrado.
  5. Use las guías para revisar entregas y tiempos.
