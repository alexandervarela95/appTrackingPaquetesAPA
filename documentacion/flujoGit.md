# Flujo Git Profesional

1. Clonar el repositorio oficial.
2. Crear ramas separadas para backend y frontend:
   - `feature/backend-api-app-tracking-envio`
   - `feature/frontend-ui-app-tracking-envio`
3. Trabajar solo el backend en la rama de backend.
4. Trabajar solo el frontend en la rama de frontend.
5. Usar `git add` y `git commit` con mensajes convencionales.
6. Hacer `git push origin <branch>` para enviar los cambios.
7. Crear pull requests separados si corresponde.

## Convenciones de commits

- `feat(backend): ...`
- `feat(frontend): ...`
- `docs(project): ...`
- `chore(git): ...`
